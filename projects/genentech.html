Genentech — AI-Powered Invoice Tracking Dashboard

Identifying a broken operational workflow inside a clinical trial finance team and independently designing and building an AI-powered solution — from problem framing to prototype to stakeholder handoff.

Organization
Roche Genentech
Year
2025–2026
Audience
Clinical trial professionals
My Role
Designer, builder & PM
The Problem

Financial analysts were receiving a plethora of invoices per day — creating chronic backlog and putting the team at risk of late payments.

Each invoice took 5–7 minutes to process manually — to open, rename, extract data, log, file, and link. At 40+ per day, that's hours of work that a well-designed system could do automatically. Nobody had framed it as a solvable problem. I did.

How I Think

Designing with constraints, not around them.

1

Frame the problem before touching the tools

I sat with the Senior FA to map the current workflow — every step, every decision point, every pain. The goal was to understand which parts were purely mechanical and which required human judgment. That distinction shaped everything.

2

Evaluate execution options rigorously

I documented three viable approaches with explicit pros and cons. Chose Streamlit + Gemini via Portkey because it was available now, fit the Google ecosystem, and could be built fast within the organization's real constraints.

3

Build iteratively with real stakeholder feedback

Biweekly 30-minute check-ins with three stakeholders — strategic direction, user feedback, and technical escalation. Between sessions I pushed updates asynchronously via a shared Google Doc project hub.

4

Know when to hand off — and how to do it well

The project paused before full integration. Rather than leaving it undocumented, I trained the manager, documented all code and next steps, and marked clearly where the project stopped and what was needed to complete it.

What I Built

A full working prototype — from email inbox to AI-powered dashboard.

Automation

Google Apps Script

Searches Gmail for invoice PDFs, downloads and saves them to organized Drive folders automatically.

AI Extraction

Gemini via Portkey / Galileo

Custom GEM reads Drive folders and extracts structured invoice data — PI name, protocol, vendor, amounts, dates.

Dashboard

Streamlit

Dashboard with filters by vendor, protocol, and PI. Overdue invoices flagged. CSV export and PDF upload built in.

Data Source

Google Drive API

Final integration point connecting the Drive folder structure to the dashboard — the step at project pause.

The Handoff

Left it better than I found it — even unfinished.

Manager training

Walked the manager through the full system — what each component does and how to run it.

Full documentation

All code, configuration, and workflow logic documented. No tribal knowledge left in my head only.

Clear next steps

Explicitly documented where the project paused and what was needed to complete and scale it.

Scalability roadmap

Outlined path to full automation with a projected 1,000+ hours saved annually.
